“单文件内容”页面设计文档

桌面端 UI 一致性 Excel 内容预览

本文档用于记录当前项目的 UI 现状、拟新增“单文件内容”按钮与页面的设计计划、当前实施进展，以及完成后的预期效果。

现状

  • 当前桌面端主页面采用 Compose Material 3 风格，整体是顶部按钮栏 + 标题说明 + 扫描操作区 + 摘要卡片 + 文件列表卡片的结构。
  • 顶部按钮栏高度固定、按钮横向排列，现有入口包括“导入”“导出”“设置”“热度”，其中“热度”属于页面级跳转入口。
  • 主体区域使用留白、卡片、分割线、标题字重区分层级，强调信息分组和阅读顺序，属于清晰、偏工具型的后台桌面 UI。
  • 文件列表目前展示的是 Excel 文件元数据，例如路径、大小、修改时间、Sheet 数和行列统计，还没有直接展示文件内容。
  • “热度”页面目前已具备独立页面跳转能力，但视觉上还是占位实现，交互模式可作为“新页面入口”的参考。

计划

  • 在顶部按钮栏中新增按钮 单文件内容，位置与“热度”同级，保留一致的按钮尺寸、排列方式和启用/禁用逻辑。
  • 当首页文件列表为空时，该按钮置灰不可点击，避免进入无数据页面。
  • 新增一个非全屏页面，延续当前标题、说明文案、卡片、间距和 Material 3 控件风格。
  • 页面中部展示当前文件的内容预览，左右两侧各放一个切换按钮，用于在文件列表中顺序浏览上一个和下一个文件。
  • 第一个文件时左侧按钮禁用，最后一个文件时右侧按钮禁用，保证切换边界清晰。
  • 文件内容读取基于当前文件列表逐个加载，优先展示 Excel 中可读的表格文本，发生读取失败时展示错误提示。

进展

已完成： 已阅读项目规约，确认需要控制单文件长度，且日志文案保持中文。

已完成： 已梳理桌面端主页面、顶部按钮栏、“热度”页面、文件列表和现有弹窗样式。

已完成： 已新增“单文件内容”页面结构、Excel 内容读取能力，以及左右切换按钮和边界禁用逻辑。

已完成： 已完成首页相关结构拆分，避免继续在超长文件中叠加页面逻辑。

效果

  • 用户在扫描完成后，可以直接从顶部入口进入“单文件内容”页面，按文件顺序逐个查看内容，无需手动定位文件。
  • 页面不会强制全屏，适合桌面端多窗口操作场景，也与当前工具型应用的使用节奏一致。
  • 左右切换操作简单直接，适合快速核对最近文件、比对不同文件内容和做连续浏览。
  • 整体视觉延续现有页面的按钮、标题、摘要卡片和内容卡片风格，新增功能不会显得割裂。
  • 如果文件列表为空或文件内容无法读取，界面会明确反馈不可用状态或错误状态，降低误操作成本。
